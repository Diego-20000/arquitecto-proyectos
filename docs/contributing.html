Contribuir al Proyecto

¡Tu ayuda es bienvenida! Aprende cómo contribuir a Arquitecto de Proyectos

Formas de Contribuir

Hay muchas maneras en las que puedes ayudar a mejorar Arquitecto de Proyectos:

  • Reportar bugs y problemas
  • Sugerir nuevas características
  • Mejorar la documentación
  • Crear plantillas/ejemplos nuevos
  • Optimizar el código existente
  • Ayudar al testing y QA
  • Compartir el proyecto con otros

Guía Para Reportar Bugs

Si encuentras un error o comportamiento inesperado, por favor reportalo creando un issue en GitHub.

Paso 1: Verifica si ya existe el issue

Busca en la lista de issues para asegurarte de que el problema aún no ha sido reportado.

Paso 2: Crea un nuevo issue

Haz clic en "New Issue" y proporciona la siguiente información:

  • Título descriptivo: Un resumen claro del problema
  • Descripción detallada: Qué esperabas que pasara y qué pasó realmente
  • Pasos para reproducir: Instrucciones exactas para reproducir el bug
  • Información del sistema: SO, versión de Node.js, etc.

Paso 3: Ejemplo de buen reporte

Título: El parser no interpreta archivos con líneas vacías Descripción: Cuando intento usar un archivo de plano que contiene líneas en blanco, el parser devuelve un error de formato inválido. Pasos para reproducir: 1. Crear archivo "test.txt" con líneas vacías 2. Ejecutar: node index.js test.txt mi-proyecto 3. Ver error en consola Sistema: - OS: Windows 10 - Node.js: v14.17.0 - npm: 6.14.13

Sugerir Nuevas Características

¿Tienes una idea para mejorar Arquitecto de Proyectos? ¡Nos encantaría escucharla!

Cómo proponer una característica

Abre un nuevo issue con el etiqueta enhancement y explica:

  • Cuál es el problema que resolvería
  • Cómo crees que debería funcionar
  • Ejemplos de uso si es posible

Configurar el Entorno de Desarrollo

1. Hacer Fork del repositorio

Ve a GitHub y haz clic en "Fork".

2. Clonar tu fork

git clone https://github.com/TU-USUARIO/arquitecto-proyectos.git cd arquitecto-proyectos

3. Añadir upstream

git remote add upstream https://github.com/Diego-20000/arquitecto-proyectos.git

4. Instalar dependencias

npm install

5. Crear una rama

git checkout -b mi-mejora

Hacer Cambios y Crear un Pull Request

Paso 1: Hacer tus cambios

Modifica los archivos según sea necesario. Asegúrate de:

  • Seguir el estilo de código existente
  • Añadir comentarios donde sea necesario
  • Probar tus cambios a fondo

Paso 2: Hacer commit

git add . git commit -m "feat: Descripción clara de tu cambio"

Usa mensajes de commit descriptivos. Ejemplos:

  • feat: Agregar soporte para archivos .json
  • fix: Corregir error en validación de nombres
  • docs: Mejorar documentación del parser
  • refactor: Simplificar lógica en builder.js

Paso 3: Push a tu fork

git push origin mi-mejora

Paso 4: Crear Pull Request

Ve a GitHub y haz clic en "Compare & Pull Request". Describe:

  • Qué cambiaste y por qué
  • Cualquier issue relacionado (#número)
  • Cómo se puede probar el cambio

Estándares de Código

JavaScript

  • Usa ESLint para validar el código
  • Usa Prettier para formatear
  • Sigue camelCase para variables y funciones
  • Usa const por defecto, let si es necesario, nunca var
  • Añade comentarios para lógica compleja

Commits

  • Commits atómicos (un cambio por commit)
  • Mensajes claros y en inglés/español consistente
  • Prefijos: feat, fix, docs, refactor, test, style
