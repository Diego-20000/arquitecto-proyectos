# Guía de Contribución

¡Agradecemos tu interés en contribuir a Arquitecto de Proyectos!

## Cómo Contribuir

### Reportar Bugs

Si encuentras un bug, por favor crea un issue en GitHub con:

1. Una descripción clara del problema
2. Pasos para reproducirlo
3. Comportamiento esperado
4. Comportamiento actual
5. Tu entorno (Sistema Operativo, versión de Node.js)

### Sugerir Mejoras

Para sugerir una mejora:

1. Usa GitHub Issues
2. Describe la mejora propuesta
3. Explica por qué sería útil
4. Lista ejemplos de casos de uso

### Pull Requests

1. Haz un fork del repositorio
2. Crea una rama para tu feature (`git checkout -b feature/AmazingFeature`)
3. Commit tus cambios (`git commit -m 'Add some AmazingFeature'`)
4. Push a la rama (`git push origin feature/AmazingFeature`)
5. Abre un Pull Request

## Estándares de Código

### JavaScript

- Usa 2 espacios de indentación
- Comentarios claros en funciones complejas
- Nombres de variables descriptivos
- Sigue la guía de eslint (`.eslintrc.json`)

### Archivos

```javascript

 * @returns {type} Descripción del retorno

  // código
}
```

## Proceso de Revisión

Todos los PRs serán revisados. Por favor:

1. Espera la revisión pacientemente
2. Responde a los comentarios
3. Realiza los cambios solicitados
4. Se requiere al menos 1 aprobación para mergear

## Código de Conducta

Por favor, sé respetuoso con otros contribuidores. Nos compromete a proporcionar un ambiente acogedor para todos.
